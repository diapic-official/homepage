プライバシーポリシー

diapic（以下、「当アプリ」）は、限られた親しい友人との間で、自身の毎日の出来事を記録し、共有することを目的とした、日記とSNSを兼ね備えたアプリです。 ユーザーがプライベートな部分を安心してさらけ出せるよう、プライバシーの侵害は決して許容されるものではありません。 当アプリにとってプライバシーの管理は最も重要な項目の一つであり、ユーザーが安心して利用できる環境を提供することをお約束します。 本プライバシーポリシーでは、当アプリが収集する情報の種類、その利用方法、情報管理の方針、ユーザーの権利について詳しく説明します。 当アプリをご利用いただく際には、本ポリシーに同意したものとみなされます。

1. お客様に関する情報

当アプリでは、ユーザーが安心してご利用いただけるよう、個人情報を適切に収集・管理します。本章では、「情報」とは個人データを含むがそれに限定されない、ユーザーに関連する全てのデータを指します。 当アプリでは、ユーザーが自主的に提供する情報（お客様が当アプリに直接提供するデータ）と、サービス利用中に当アプリが自動的に収集する情報（お客様が当アプリを利用する際に収集されるデータ）の両方を取り扱います。

1.1 お客様が自主的に提供するデータ

  • アカウント作成データ

    アカウント作成時に提供が必要な基本情報です。これらの情報がない場合、当アプリのサービスを提供することができません。

    • 必須データ: メールアドレス、暗号化されたパスワード、ユーザーID
    • 任意データ: ユーザーネーム、生年月日、性別、自己紹介文、プロフィール写真

  • コンテンツデータ

    当アプリで提供されるコンテンツや投稿関連のデータを指します。

    • プロフィール写真: ユーザーが設定するプロフィール画像
    • 投稿関連データ: 投稿本文、画像、感情タグ、対象タグ、リアクションなど

  • インタラクションデータ

    当アプリ内での他のユーザーとの関わりに関連するデータです。

    • コメント: 投稿に対するテキストベースの返信として提供される情報
    • チャット: 個別またはグループ間で送信されるダイレクトメッセージ
    • リアクション: 投稿やコメントに対する反応

  • お問い合わせおよびサポート

    お客様が当アプリのサポートへ連絡した際に提供される情報です。

    • メールやメッセージの内容
    • 必要に応じて提出される身分証明書やその他の関連情報

1.2 サービス利用時に自動的に収集されるデータ

  • 接続および技術データ

    当アプリのサービスを利用するデバイスやネットワーク接続に関連する情報です。

    • IPアドレス
    • アカウント作成日、最終ログイン日
    • 使用しているデバイスの種類、OSのバージョン、言語設定
    • システム構成データ、イベントログ

  • 使用状況データ

    当アプリの利用履歴やアクションに基づく情報です。

    • 投稿の編集ログ
    • 他のユーザーとのコメントやチャット
    • 閲覧したコンテンツやその閲覧頻度

  • 交流データ

    他のユーザーとのやり取りやネットワーク上の関係に関するデータです。

    • フレンド数、フレンドリクエスト、最も交流のあるユーザー
    • コメントやチャットの頻度や内容
    • 投稿に対するコメントやリアクション

  • 装置データ

    お客様が使用するデバイスに関する情報です。

    • デバイスの種類、OSバージョン、アプリケーションのクラッシュログ

2. お客様に関する情報の使用方法

当アプリでは、ユーザーのプライバシーを最優先に考え、ユーザーの情報を慎重かつ責任を持って取り扱います。 私たちは、ユーザーの情報を第三者に販売することは一切ありません。 ユーザーにより良いサービスを提供するため、情報はさまざまな目的で使用されます。 本章では、当アプリが情報をどのように使用するかについて、具体的なケースを以下に分類して説明します。

2.1 当アプリの運営，改善，および提供

ユーザーが当アプリを快適に利用できるよう、以下の目的で情報を使用します。

  • アカウント作成、コンテンツの保存・表示、投稿やプロフィール情報の管理
  • ユーザーが関心を持ちそうな投稿やアカウントを推奨
  • サービスが正常に動作していることの確認と機能の改善
  • 新機能の開発と既存機能の最適化

2.2 安全とセキュリティ

ユーザーおよび当アプリ全体の安全を確保するため、以下のように情報を利用します。

  • 不正アクセスやアカウント乗っ取りなどのセキュリティリスクの検出と防止
  • ユーザー間の不適切なコンテンツや行動の監視および対応
  • 法令や規制の遵守のための確認と報告

2.3 パフォーマンス測定と分析

当アプリのパフォーマンスを向上させるため、以下の目的で情報を使用します。

  • アプリのクラッシュ原因や技術的な問題の特定と解決
  • アプリの安定性や機能性を向上させるための分析
  • 投稿やリアクションなどの利用状況の測定を通じた、アカウントやコンテンツの適切な推奨

2.4 ユーザーとのコミュニケーション

当アプリでは、ユーザーとの双方向のコミュニケーションを重要視しています。そのため、以下のような場面で情報を使用します。

  • サービスの機能や更新に関する通知
  • ユーザーからのお問い合わせやサポートリクエストへの対応
  • サービスの運営や利用に関する重要な連絡

2.5 ユーザー体験の研究と改善

当アプリは、ユーザーの声を直接取り入れることで、より良いサービスの開発を目指しています。

  • ユーザーのフィードバックに基づいた機能改善
  • ユーザー参加型の調査や製品テストの実施（事前に参加可否を確認）
  • 調査結果を基にした新しい機能やサービスの試験運用

2.6 広告やマーケティング

現時点で当アプリでは広告を表示していませんが、将来的に広告を導入する際には、以下の方針に従います。

  • ユーザーの興味や関心に基づいたコンテンツの提案
  • 広告サービスを提供するためのデータの限定的な活用（ユーザーへの通知後に実施）

当アプリでは、情報を慎重に管理し、目的外の利用を防ぐために厳格な基準を設けています。 利用目的の変更が必要になった場合は、事前に通知し、ユーザーの同意を取得します。

3. 情報の管理方法

当アプリでは、ユーザーの個人情報を適切に保護し、安心してサービスをご利用いただけるよう、情報管理とユーザーの権利に最大限の配慮をしています。 当アプリの情報管理は、外部サービスである Firebase（Google LLC） に一任しており、Firebaseが提供する高度なセキュリティ技術と基準に基づいて運用されています。 さらに、ユーザーは自身の情報に対して以下の権利を有し、当アプリではこれらの権利を尊重し実現するための仕組みを提供します。

3.1 Firebaseによる情報管理

  • データの暗号化: 保存時および通信時に暗号化を適用し、外部からの不正アクセスを防止します。
  • アクセス制御: アクセス権限を厳格に制限し、認証されたユーザーや管理者のみがデータにアクセス可能です。
  • セキュリティ更新: 最新のセキュリティ技術を適用し、データ保護の継続的な向上を図ります。

3.2 保存期間

当アプリでは、ユーザーが投稿したデータや関連情報を無制限に保存します。これは、当アプリが日記とSNSの特性を併せ持ち、 ユーザーが自身の記録を自由に振り返ることができる環境を提供するためです。また、保存されたデータは以下のような目的にも活用されます。

  • AI解析を通じたユーザーの興味関心や行動パターンの把握
  • 将来設計のヒント提供や自己分析ツールとしての活用

3.3 ユーザーの権利と情報管理

  • アカウントの削除

    ユーザーは、アカウント削除をリクエストすることで、以下の権利を行使できます。

    • アカウントに紐づく個人情報や投稿データの削除
    • データ削除が完了した後のサービス利用停止

    なお、法的要件やセキュリティ上の理由により、必要最低限の情報を保持する場合があります。


  • データの確認および修正

    ユーザーは、自身のプロフィール情報や投稿データを自由に確認・修正する権利を有します。これには以下が含まれます。

    • プロフィール写真や自己紹介文の更新
    • 過去の投稿内容やタグの編集

  • 利用停止のリクエスト

    利用停止を希望する場合は、アカウント削除申請を行う必要があります。 削除申請により、今後のサービス利用が停止され、保存データの管理が適切に行われます。

  • データの保存および削除状況の透明性

    ユーザーは、自身のデータがどのように保存され、削除されるかについて説明を求める権利を有します。 当アプリでは、Firebaseを通じたデータ管理の透明性を維持し、ユーザーからの問い合わせに適切に対応します。

3.4 情報外部提供ポリシーの参照

当アプリでの情報管理の詳細については、情報外部提供ポリシー をご参照ください。 本ポリシーには、Firebaseを含む外部サービスに情報を提供する際の管理基準が記載されています。

当アプリでは、情報管理の透明性を高めるとともに、ユーザーの権利を最大限尊重する仕組みを提供します。 情報管理に関するご質問や権利行使のご要望がある場合は、サポート窓口までご連絡ください。

4. お客様に関する情報の共有

当アプリでは、ユーザーの情報を慎重かつ責任を持って管理し、以下の場合を除き、第三者と情報を共有することはありません。 情報共有が必要な場合には、適切な基準に基づき、最小限の範囲で実施します。また、共有される情報がユーザーのプライバシーを侵害しないよう徹底した配慮を行います。

4.1 サービス提供のための共有

Firebaseへの共有

当アプリのデータ管理および技術インフラは、すべてFirebaseによって運用されています。以下のようなサービスを提供しています。

  • データのホスティングおよび保存: 投稿データ、ユーザー情報、プロフィール写真、画像ファイルなどの保存
  • 認証機能: メールアドレスおよび暗号化されたパスワードを利用した安全なユーザー認証
  • 通知機能: ユーザーへの重要な通知の送信
  • セキュリティ対応: 不正アクセスの検知と防止、データ保護の強化

Firebaseに情報を提供することで、当アプリは高いセキュリティ基準を維持し、安定したサービスを提供しています。 詳細については、情報外部提供ポリシー をご参照ください。

ユーザー間の共有

ユーザーが当アプリを通じて投稿コンテンツ（本文、画像、感情タグ、対象タグなど）を共有する場合、そのコンテンツは指定した公開範囲（公開、限定公開、非公開）に応じて他のユーザーに表示されます。

  • 公開: 全ユーザーが閲覧可能です。
  • 限定公開: 指定された友人やグループ内のユーザーにのみ表示されます。
  • 非公開: 自身のみが閲覧可能です。

4.2 法律上の要請およびセキュリティ対応

法的要件を満たすため、以下の場合に情報を共有することがあります。

  • 法令に基づく正式な要請を受けた場合
  • 詐欺、不正行為、セキュリティ上の問題への対応が必要な場合
  • 他のユーザーや社会全体の安全を確保するために必要な場合

これらの状況においても、共有する情報の範囲は最小限に留めます。

4.3 オーナーシップの変更

当アプリが組織再編、合併、資産売却、または事業譲渡を行う場合、ユーザー情報が新しいオーナーに引き継がれることがあります。 その場合でも、本プライバシーポリシーの内容が引き続き適用されます。

4.4 ユーザーの選択に基づく共有

ユーザーが特定の外部サービスとの連携や統合機能を利用する場合、その連携先サービスに情報が共有されることがあります。

  • ユーザーが外部アプリケーションとのデータ連携を許可した場合
  • 他のサービスで当アプリのデータを利用する機能を有効化した場合

このような共有は、ユーザーの同意に基づいて行われます。

当アプリでは、Firebase（Google LLC）のみを利用した情報管理を徹底し、その他のパートナーに情報を共有することはありません。 情報共有についてさらに詳しい情報が必要な場合やご不明な点がある場合は、サポート窓口までお問い合わせください。

5. プライバシーポリシーの変更

当アプリは、サービスの向上や法令改正に伴い、本ポリシーを変更する場合があります。 変更が行われた場合、アプリ内またはお使いのストアプラットフォームで通知します。 重要な変更については、改めて同意を求める場合があります。


情報外部提供ポリシー

diapic（以下、「当アプリ」）では、ユーザーの皆様に安心してサービスをご利用いただけるよう、適切なデータ管理を行うことを重要視しています。本ポリシーでは、当アプリが利用する外部サービスへの情報提供について、その必要性や提供される情報の内容、目的、データ保護方法について明確に説明します。
外部サービスへの情報提供は、当アプリの正常な運営において必須であり、この提供に同意いただけない場合、一部またはすべての機能が利用できなくなる場合があります。

1. 提供先

当アプリでは、以下の外部サービスを利用しています。

  • Firebase Authentication（Google LLC）: ユーザー認証およびアカウント管理の基盤を提供。
  • Firestore Database（Google LLC）: アプリ内で管理される各種データ（ユーザー情報、投稿情報、インタラクション情報、設定情報など）を保存。
  • Firebase Storage（Google LLC）: 画像やその他メディアファイルを安全に保存および管理。

2. 提供される情報と目的

1. Firebase Authentication

提供される情報: メールアドレス / 暗号化されたパスワード
目的: Firebase Authenticationを利用することで、安全かつ信頼性の高いユーザー認証を実現します。この仕組みは、ログイン機能やアカウント管理の基盤を支え、ユーザーがスムーズにサービスを利用できる環境を提供します。

2. Firestore Database

Firestore Databaseは、アプリの主要データを管理するために使用されています。

  • ユーザー情報: メールアドレス / 暗号化されたパスワード / ユーザーネーム / ユーザーID / プロフィール写真のURL / 自己紹介文 / 生年月日 / 性別
  • 投稿情報: 投稿本文 / 感情タグ / 対象タグ / 公開範囲（公開 / 非公開 / 限定公開）
  • インタラクション情報: フレンドリスト / いいね情報 / ダイレクトメッセージ（コメントやチャット） / 通知情報
  • アプリ設定情報: ユーザーごとの通知設定やプライバシー設定（JSON形式など）

3. Firebase Storage

提供される情報: 投稿に関連する画像データ / その他メディアファイル
目的: 画像やメディアファイルを安全に保存・管理し、投稿やプロフィール画像として使用します。これにより、高品質なメディア体験を提供します。

3. 提供の必要性

外部サービスへの情報提供は、当アプリの基本機能（ログイン、投稿、通知、設定など）を実現するために不可欠です。これらの情報が提供されない場合、以下の機能が制限されます。

  • Firebase Authenticationに情報が提供されない場合: ユーザー認証やログイン機能が利用できません。
  • Firestore Databaseに情報が提供されない場合: 投稿やユーザー間のインタラクション、設定情報の保存・管理ができません。
  • Firebase Storageに情報が提供されない場合: 画像やメディアファイルの保存・管理が行えません。

4. データ保護

外部サービスに提供される情報は、Google LLCが提供するセキュリティ基準およびプライバシーポリシーに基づいて、以下の方法で保護されています。

  • 暗号化: 保存時および転送時のデータは暗号化され、不正アクセスから保護されています。
  • アクセス制御: 各サービスへのアクセスは、アプリケーションで設定されたセキュリティルールによって厳格に管理されています。
Googleのプライバシーポリシーに関する詳細は、こちらをご確認ください。
